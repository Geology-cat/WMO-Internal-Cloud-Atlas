# WMO 国際雲図帳 日本語訳 — ビルドガイド

WMO (世界気象機関) の [International Cloud Atlas](https://cloudatlas.wmo.int/en/home.html) を日本語に翻訳した静的Webサイトです。  
コンテンツは **Markdown (.md)** で管理し、Python スクリプトで **HTML** に一括変換します。

## ディレクトリ構成

```
日本語訳/
├── build.py                  # ビルドスクリプト（MD → HTML 変換）
├── requirements.txt          # Python 依存パッケージ
├── style.css                 # 共通スタイルシート
├── index.md                  # トップページ（Markdown原本）
├── index.html                # ← build.py が生成
├── ica-home ja.md            # ホームページ本文
├── templates/
│   └── base.html             # 共通HTMLテンプレート（Jinja2）
├── Cirrus ja/                # 各雲形のページ（サブディレクトリ）
│   ├── Cirrus.md             #   Markdown 原本
│   ├── Cirrus.html           #   ← build.py が生成
│   ├── Ci_fib_unc.jpg        #   画像（md から相対パスで参照）
│   └── ...
├── Cumulus ja/
├── Clouds ja/
├── Other Meteors ja/
├── Observing Clouds ja/
├── 翻訳版作成にあたって/
├── 付録/
├── ... (その他の雲形ディレクトリ)
└── README.md                 # このファイル
```

**ポイント:**
- `.md` ファイルがコンテンツの原本。**直接編集するのは常に `.md` のみ**。
- `.html` ファイルは `build.py` が自動生成する。手動で編集しない。
- 画像は各 `.md` と同じディレクトリに配置し、md 内で相対パスで参照する。

## セットアップ

### 前提条件

- **Python 3.8 以上** がインストールされていること

### インストール

```bash
# プロジェクトディレクトリに移動
cd /path/to/日本語訳

# 依存パッケージをインストール
pip3 install -r requirements.txt
```

`requirements.txt` に含まれるパッケージ:
- **markdown** — Markdown → HTML 変換ライブラリ
- **Jinja2** — HTMLテンプレートエンジン

---

## 基本コマンド

すべてのコマンドはプロジェクトルート（`日本語訳/`）で実行してください。

### 全ページをビルド（MD → HTML 変換）

```bash
python3 build.py
```

ディレクトリ内の全 `.md` ファイルを走査し、同名の `.html` を生成します。  
既存の `.html` は上書きされます。

### 生成した HTML を一括削除

```bash
python3 build.py --clean
```

`.md` に対応する `.html` ファイルのみを削除します。  
元の `.md` ファイルや画像は一切削除されません。

### ローカルでプレビュー

```bash
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080/index.html` を開いて表示を確認できます。  
終了は `Ctrl+C` です。

## ユースケース別の操作手順

### 1. 既存ページの内容を修正する

> 例: 巻雲のページの説明文を修正したい

**手順:**

1. 対象の `.md` ファイルを編集する
   ```
   Cirrus ja/Cirrus.md   ← テキストエディタで編集
   ```

2. ビルドスクリプトを実行する
   ```bash
   python3 build.py
   ```

3. 確認する
   ```bash
   python3 -m http.server 8080
   # ブラウザで http://localhost:8080/Cirrus%20ja/Cirrus.html を確認
   ```

**これだけで完了です。** テンプレートや CSS の変更は不要です。

---

### 2. 新しいページを追加する

> 例: 「新しい雲の種類」のページを追加したい

**手順:**

#### Step 1: Markdown ファイルを作成する

サブディレクトリを作り、その中に `.md` ファイルを作成します。

```
NewCloud ja/
├── NewCloud ja.md    ← 新規作成
└── photo.jpg         ← 使用する画像があれば配置
```

`.md` ファイルの先頭は `# 見出し` で始めてください。  
この見出しがHTMLの `<title>` に使用されます。

```markdown
# 新しい雲の種類

## 定義
ここに定義を書く...

## 特徴
ここに特徴を書く...

<div align="center">
  <img src="photo.jpg" alt="写真の説明" width="800">
  <p><strong>写真の説明</strong></p>
</div>
```

#### Step 2: index.md にリンクを追加する

`index.md` を開き、適切な場所にリンクを追加します。

```markdown
- [新しい雲の種類](NewCloud%20ja/NewCloud%20ja.md)
```

> **注意:** リンク先は `.md` で記述してください。ビルド時に自動的に `.html` に変換されます。  
> ディレクトリ名にスペースが含まれる場合は `%20` でエンコードしてください。

#### Step 3: ビルドする

```bash
python3 build.py
```

新しいページが自動的に検出され、HTML が生成されます。  
`index.html` のリンクも `.html` に自動的に書き換わります。

---

### 3. 共通部分（ヘッダー・ナビ・フッター）を変更する

> 例: フッターの文言を変更したい、ヘッダーのタイトルを変えたい

共通部分は **テンプレートファイル** で一元管理されています。

**対象ファイル:** `templates/base.html`

**手順:**

1. `templates/base.html` を編集する

   ```html
   <!-- ヘッダーのタイトルを変更する場合 -->
   <header>
       <div class="header-inner">
           <h1>ここを変更</h1>
           <p class="subtitle">ここも変更可能</p>
       </div>
   </header>

   ...

   <!-- フッターの文言を変更する場合 -->
   <footer>
       <p>ここを変更</p>
   </footer>
   ```

2. 全ページを再ビルドする
   ```bash
   python3 build.py
   ```

**1回の編集で全18ページに反映されます。**

#### テンプレートの構造

```
base.html
├── <head>        … meta タグ, CSS リンク, タイトル
├── <header>      … サイトタイトル, サブタイトル
├── <nav>         … ナビゲーションメニュー
├── <main>        … {{ content }}（各ページの本文がここに挿入される）
└── <footer>      … フッター
```

### 4. ナビゲーションメニュー項目を変更する

> 例: ナビに「用語集」というリンクを追加したい

ナビメニューは **`index.md` のヘッダーメニュー部分から自動生成** されます。  
`index.md` を編集するだけで、テンプレートやスクリプトの変更は一切不要です。

**手順:**

1. `index.md` の `---` で囲まれたヘッダーメニュー部分を編集する

   ```markdown
   ---------------------------------------------
   ヘッダーメニュー
   - [トップページ](index.md)
   - [翻訳版作成にあたって](翻訳版作成にあたって/翻訳版作成にあたって.md)
   - [付録](付録/付録.md)
   - [用語集](用語集/用語集.md)            ← 追加したい項目
   - [元サイト](https://cloudatlas.wmo.int/en/home.html)
   ---------------------------------------------
   ```

   - `- [ラベル](URL)` の形式で記述します
   - 内部リンクは `.md` 拡張子で書く（ビルド時に自動で `.html` に変換）
   - 外部リンク (`https://...`) はそのまま記述（自動で `target="_blank"` が付与されます）
   - 項目の順序を入れ替えれば、ナビの表示順も変わります
   - 項目を削除すれば、ナビからも消えます

2. ビルドする
   ```bash
   python3 build.py
   ```

**これだけで全ページのナビが更新されます。**

---

### 5. CSS（デザイン）を変更する

> 例: 見出しの色を変えたい、フォントを変えたい

**対象ファイル:** `style.css`

**手順:**

1. `style.css` を編集する

2. **再ビルドは不要** — CSS は HTML から相対パスで参照されているため、  
   CSS の変更はブラウザをリロードするだけで即座に反映されます。

#### 主要な CSS カスタム変数（`:root` で定義）

| 変数名 | 用途 | デフォルト値 |
|--------|------|-------------|
| `--sky-light` | 背景色（薄い空色） | `#ddeeff` |
| `--sky-mid` | ナビホバー、見出し下線 | `#a8cce8` |
| `--sky-deep` | テーブルヘッダ等 | `#5590b8` |
| `--accent` | リンク色、h3色 | `#2a6496` |
| `--text-dark` | 本文テキスト色 | `#1a2a3a` |
| `--width` | コンテンツ幅 | `80%` |
| `--max-width` | 最大幅 | `1200px` |

これらの変数を変更するだけで、サイト全体の色調を一括で変更できます。

## ビルドスクリプトの仕組み

`build.py` は以下の処理を順番に実行します。

```
1. index.md のヘッダーメニュー部分 (--- で囲まれた領域) をパース
   → ナビゲーション項目のリストを生成
       ↓
2. プロジェクトルート以下の全 .md ファイルを再帰的に取得
       ↓
3. 除外対象をスキップ
   - templates/ ディレクトリ内
   - .windsurf/ 等の隠しディレクトリ内
   - EXCLUDE_FILES に指定されたファイル（変換プロンプト.txt, README.md）
       ↓
4. 各 .md ファイルに対して:
   a. Markdown テキストを読み込む
   b. index.md の場合 → ヘッダーメニュー部分を本文から除去
   c. 先頭の # 見出しを <title> 用のタイトルとして抽出
   d. Python-Markdown で HTML に変換（tables, fenced_code, toc 等の拡張を使用）
   e. 変換後 HTML 内の .md リンクを .html に一括置換
   f. 出力先ディレクトリの深さに応じて CSS パス・ナビリンクの相対パスを計算
   g. ナビ項目の active フラグを判定（現在ページとナビリンク先が一致するか）
   h. Jinja2 テンプレート (base.html) にコンテンツとナビ項目を流し込み
   i. 同名の .html ファイルとして書き出す
```

### 変換対象から除外したいファイルを追加する

`build.py` の `EXCLUDE_FILES` にファイル名を追加してください:

```python
EXCLUDE_FILES = {"変換プロンプト.txt", "README.md", "DRAFT.md"}
```

## テンプレート変数一覧

`templates/base.html` 内で使用できる Jinja2 変数の一覧です。

| 変数名 | 型 | 説明 |
|--------|-----|------|
| `{{ title }}` | str | ページタイトル（md の最初の `# 見出し` から自動抽出） |
| `{{ content }}` | str | md から変換された HTML 本文 |
| `{{ css_path }}` | str | そのページから見た `style.css` への相対パス |
| `{{ nav_items }}` | list | ナビゲーション項目のリスト（`index.md` から自動生成） |
| `{{ is_index }}` | bool | 現在のページが `index.html` かどうか |

### `nav_items` の各項目のプロパティ

| プロパティ | 型 | 説明 |
|------------|-----|------|
| `item.label` | str | メニューの表示ラベル（例: 「トップページ」） |
| `item.url` | str | 現在ページからの相対パス（自動計算済み） |
| `item.external` | bool | 外部リンクかどうか |
| `item.active` | bool | 現在のページがこのナビ項目に対応するか |

### テンプレートでの使用例

```html
<!-- ナビゲーションのループ出力 (現在の base.html で使用中) -->
{% for item in nav_items %}
<li>
    {% if item.external %}
    <a href="{{ item.url }}" target="_blank">{{ item.label }} ↗</a>
    {% else %}
    <a href="{{ item.url }}"{% if item.active %} class="active"{% endif %}>{{ item.label }}</a>
    {% endif %}
</li>
{% endfor %}

<!-- index.html だけタイトルを変える -->
{% if is_index %}短いタイトル{% else %}{{ title }} | サイト名{% endif %}
```

## 注意事項・トラブルシューティング

### md 内のリンクは `.md` で書く

```markdown
<!-- ✅ 正しい — build.py が自動的に .html に変換する -->
[巻雲](Cirrus%20ja/Cirrus.md)

<!-- ❌ 避ける — md としてのプレビューが壊れる -->
[巻雲](Cirrus%20ja/Cirrus.html)
```

### ディレクトリ名にスペースを含む場合

md 内のリンクでは `%20` でエンコードしてください:

```markdown
[ホーム](ica-home%20ja.md)
[巻雲](Cirrus%20ja/Cirrus.md)
```

### 画像の配置

画像は対応する `.md` と **同じディレクトリ** に配置してください。  
md 内では相対パスで参照します:

```markdown
<img src="photo.jpg" alt="説明">
```

### ビルドしても変更が反映されない場合

1. ブラウザのキャッシュをクリアしてリロード（`Cmd+Shift+R` / `Ctrl+Shift+R`）
2. `python3 build.py --clean` で一度 HTML を全削除してから再ビルド:
   ```bash
   python3 build.py --clean
   python3 build.py
   ```

### noindex について

全ページの `<head>` に以下のメタタグが自動挿入されます:

```html
<meta name="robots" content="noindex, nofollow">
```

これにより、Google 等の検索エンジンにインデックスされません。  
この設定を変更したい場合は `templates/base.html` の該当行を編集してください。

### index.md のヘッダーメニュー部分

`index.md` にある `---` で囲まれた領域は **ナビゲーションメニューの定義** です:  

```markdown
---------------------------------------------
ヘッダーメニュー
- [トップページ](index.md)
- [翻訳版作成にあたって](翻訳版作成にあたって/翻訳版作成にあたって.md)
- [付録](付録/付録.md)
- [元サイト](https://cloudatlas.wmo.int/en/home.html)
---------------------------------------------
```

`build.py` はこの部分をビルド時に:
1. **パースしてナビゲーションメニューを自動生成** します
2. **`index.html` の本文からは自動的に除去** します（二重表示防止）

ナビの追加・変更・削除はこのヘッダーメニュー部分を編集するだけで完結します。

## クイックリファレンス

| やりたいこと | 編集するファイル | ビルド要否 |
|-------------|-----------------|-----------|
| ページの文章を修正 | 対象の `.md` | **要** (`python3 build.py`) |
| 新しいページを追加 | 新規 `.md` + `index.md` にリンク追加 | **要** |

| 生成HTML全削除 | — | `python3 build.py --clean` |
